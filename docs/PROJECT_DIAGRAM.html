Что происходит при одном сообщении

Функция chat() в assistant_service.py

Сохранить вопрос пользователя

save_message(db, 'user', text) → таблица messages

Загрузить историю

get_last_messages(db, 10) — последние 10 реплик

Если история ≥ 10 — сжать память

6 старых сообщений → Ollama извлекает факты (JSON) → user_facts + Chroma → удаление старых messages. Второй вызов LLM!

Собрать контекст

Факты SQLite + summary + история + RAG-поиск по вопросу → build_context()

Сгенерировать ответ

generate_response() → POST Ollama /api/chat

Сохранить и вернуть

save_message(db, 'assistant', answer) → ответ в CLI / Web

messages

  • role user / assistant
  • content текст
  • Живёт пока не сожмут

user_facts + Chroma

  • key name, goal, …
  • value строка
  • RAG ищет по смыслу

summaries

  • Только если JSON сломался
  • Запасной контекст

Таблицы SQLite (assistant.db)

Файл backend/app/database/models.py

messages

  • id PK
  • role string
  • content text
  • timestamp UTC

user_facts

  • id PK
  • key name, goal…
  • value text
  • created_at

summaries

  • id PK
  • content text
  • created_at

Операции — repository.py: save_message, get_last_messages, save_user_fact, get_all_user_facts, save_summary, get_lastest_summary
